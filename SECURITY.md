# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in GrowthLens, please report it responsibly.

**Email:** richard@richardgoold.com

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fix (optional)

## Response Timeline

- **Acknowledgement:** Within 48 hours
- **Initial assessment:** Within 5 business days
- **Resolution target:** Depends on severity (Critical: 1 week, High: 2 weeks, Medium: 1 month, Low: 3 months)

## Scope

This policy applies to the GrowthLens web application at growthlens.app and the associated GitHub repository.

## Out of Scope

- Social engineering attacks against GrowthLens staff
- Denial of service attacks
- Issues in third-party services (Supabase, GitHub Pages, Resend)

## Responsible Disclosure

We ask that you do not publicly disclose the vulnerability until we have had a reasonable opportunity to address it. We will credit reporters in our changelog unless anonymity is requested.
